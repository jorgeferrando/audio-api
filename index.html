Architecture

Clean Architecture with four layers. Dependencies flow inward: presentation depends on application, application depends on domain, and infrastructure implements ports defined by domain and application. The composition root (index.ts) is the only file that knows all layers. 

presentation/     Controllers, routes, middlewares (auth, validation, error handling)
application/      Use cases, DTOs, application ports (IFileStorage, IJobPublisher)
domain/           Entities (AudioTrack, ProcessingJob), repository ports
infrastructure/   MongoDB, Redis, RabbitMQ, MinIO, Express, ffmpeg, nginx, Winston

Request Flow

Upload:   Client ──► nginx ──► multer ──► ffprobe (validate) ──► MinIO (stream) ──► MongoDB + RabbitMQ
Worker:   RabbitMQ ──► Consumer ──► MinIO (download) ──► ffmpeg ──► MinIO (upload) ──► MongoDB + Redis
Status:   Client ──► nginx ──► Redis (cache hit?) or MongoDB ──► cache ──► DTO
Download: Client ──► nginx ──► MinIO stream

Key Patterns

Result/Either Monad

No throw in domain or application layers. Every operation returns Result<T, E> — the caller explicitly handles success and error paths. Early-return pattern keeps the happy path readable.

Port & Adapter

Every infrastructure dependency (MongoDB, Redis, RabbitMQ, MinIO, ffmpeg) is behind an interface. The domain never imports infrastructure. Swapping MinIO for GCS is a single-file change.

Saga Compensation

When a multi-entity operation fails mid-processing, compensation logic reconstitutes entities in their last persisted state and marks them as FAILED. No distributed transactions needed.

Graceful Drain

On SIGTERM, the worker stops accepting new messages (channel.cancel()) and waits for the in-flight job to finish before closing connections. Zero lost work during K8s rolling updates.

Project Evolution

The project went through 7 rounds of technical review. Each round identified defects, and each defect was addressed in the next iteration. This timeline shows how the project evolved from a basic CRUD to a production-grade system.

Phase 1 — Foundation

Domain Entities + TDD

AudioTrack and ProcessingJob with state machines, Result/Either pattern, native # private fields, static create() factories. 37 tests. Co-located test files.

Phase 2 — Infrastructure

MongoDB, Redis, RabbitMQ

Repository implementations with Mongoose, RedisCacheService with TTL strategy, RabbitMQ publisher/consumer with DLQ. Integration tests with mongodb-memory-server. Grew to 109 tests.

Phase 3 — HTTP Layer

Express + Zod + Supertest

AudioController with Zod validation, error handler mapping AppError codes to HTTP status, health endpoint. HTTP integration tests with Supertest. 129 tests.

Phase 4 — Real Audio

File Upload + ffmpeg Processing

Multer for multipart uploads, ffmpeg for real audio effects (reverb, echo, pitch shift), download endpoint. Docker Compose with 5 services. Full flow tested end-to-end.

Phase 5 — Review Round 1

External Review: 10 Defects Found

Critical: local file storage incompatible with K8s multi-replica. Important: "Microservices" naming incorrect, API key without scoping. All addressed: MinIO migration, naming fix, API key auth, graceful shutdown, rate limiting.

Phase 6 — MinIO + K8s

Object Storage + Kubernetes Deployment

Migrated from local disk to MinIO (S3-compatible). IFileStorage port enables GCS migration. K8s manifests with separate API/worker Deployments, tested with Docker Desktop K8s. One-button deploy and test scripts.

Phase 7 — Hardening

nginx, CSP, Streaming, Validation

nginx reverse proxy (Express API-only), CSP compliance (no inline styles/scripts), streaming upload to MinIO, ffprobe content validation, readiness probe with dependency checks, frontend modernization (ES modules, AbortController, oklch, CSS nesting). 176 tests, 11 ADRs.

Challenges Overcome

Split-Brain File Storage

Problem: Local disk storage + multiple K8s replicas = download failures
Solution: MinIO object storage with IFileStorage port
All pods read/write from the same MinIO instance. The port allows migrating to GCS/S3 by changing one implementation file.

CSP Blocking Inline Styles

Problem: style="display:none" silently ignored by Helmet CSP
Solution: .hidden CSS class + show()/hide() utility
Helmet's style-src 'self' blocks all inline styles without warning. Discovered after hours of debugging why elements were visible despite having display:none in the HTML.

Saga Compensation with Private Fields

Problem: Entity already mutated to COMPLETED in memory, can't transition to FAILED
Solution: reconstitute() creates fresh entities in PROCESSING state
When a DB save fails after calling job.complete(), the state machine rejects COMPLETED → FAILED. Reconstituting from the last persisted state (PROCESSING) allows the FAILED transition.

50MB Upload Without RAM Buffer

Problem: multer memoryStorage holds entire file in RAM per request
Solution: multer diskStorage → createReadStream → MinIO stream
Files land in os.tmpdir(), get streamed to MinIO, and cleaned up. No 50MB buffer in memory. Worker does the same: download from MinIO to temp, process with ffmpeg, upload result, cleanup in finally.

Clean Architecture Violations

Problem: Controller importing repository directly (skipping application layer)
Solution: Extracted DownloadAudioUseCase to respect layer boundaries
Caught during code review. The download endpoint was calling audioRepo.findById() from the controller. Refactored to a proper use case that the controller delegates to.

Graceful Shutdown During Processing

Problem: SIGTERM kills worker mid-ffmpeg, job stuck in PROCESSING forever
Solution: consumer.stop() cancels consumer + waits for in-flight job
The consumer's stop() method calls channel.cancel() and returns a Promise that resolves only when the current job finishes. Tested with a concurrent test that verifies stop() doesn't resolve until the job completes.

Architecture Decision Records

Every non-obvious decision is documented with context, rationale, and consequences. The ADRs answer "why" — the code answers "what".

  • 001Result/Either Pattern for Error Handling — no throw in domain/application
  • 002Clean Architecture with Four Layers — dependency direction enforced
  • 003Co-located Unit Tests, Separated Integration Tests
  • 004ILogger as Port, Multiple Adapters (Winston + Console)
  • 005Cache in the Use Case, Not the Repository — TTL depends on business logic
  • 006Saga Compensation Pattern — reconstitute entities for failure recovery
  • 007RabbitMQ Prefetch 1, Nack Without Requeue — DLQ recovery strategy
  • 008MinIO Object Storage — eliminates split-brain in multi-replica K8s
  • 009Contract Tests Only for Ports with Multiple Implementations (YAGNI)
  • 010Nginx Reverse Proxy — Express is API-only, nginx serves static files
  • 011TLS in Production — cert-manager + Ingress (documented, not implemented)
